# Проверка страницы 4

## Сценарий локальной проверки

1. Запустить backend: `cd server && npm install && npm run dev`.
2. Запустить web: `cd mobile && npm install && npm run web`.
3. Зарегистрировать новый личный аккаунт.
4. Открыть `Профиль`.
5. Нажать `☰`.
6. Нажать `Создать бизнес`.
7. Заполнить название, никнейм, категорию и дополнительные поля.
8. Нажать `Создать бизнес`.
9. Проверить, что открывается бизнес-профиль.
10. Нажать имя аккаунта `▼`.
11. Переключиться обратно на личный аккаунт.
12. Проверить, что профиль снова личный.
13. Переключиться на бизнес-аккаунт и проверить, что профиль снова бизнес.

## Что должно работать

- Создание бизнес-аккаунта отдельным аккаунтом.
- Автоматическое переключение `activeAccountId` на созданный бизнес.
- Переключение между личным и бизнес-аккаунтом.
- Нижнее меню остаётся `Главная / Видео / + / Карта / Профиль`.
- `+` не создаёт бизнес и не создаёт пост напрямую.

## Что не должно появиться

- `user.isBusiness`.
- Создание бизнеса на экране регистрации.
- Бизнес вместо личного аккаунта.
- Вкладка сообщений в нижнем меню.
- Кнопка создания контента на карте.
