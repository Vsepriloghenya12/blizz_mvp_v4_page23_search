# Страница / модуль 12 — Главная: Лента и Витрина

## Назначение

Главная у пользователя делится на два режима внутри одного экрана:

```text
Главная
├── Лента
└── Витрина
```

`Лента` — обычные посты личных аккаунтов.  
`Витрина` — бизнес-контент: бизнес-посты и предложения.

Витрина не является отдельной вкладкой нижнего меню. Нижнее меню остаётся:

```text
Главная / Видео / + / Карта / Профиль
```

## Общая схема

```text
┌────────────────────────────────────┐
│ Близз                         💬   │
│                                    │
│ Близзы:                            │
│ [Ваш Близз +] [Аккаунт] [Бизнес]   │
│                                    │
│ [Лента] [Витрина]                  │
│                                    │
│ Если Лента:                        │
│   посты личных аккаунтов           │
│                                    │
│ Если Витрина:                      │
│   бизнес-посты                     │
│   предложения бизнеса              │
└────────────────────────────────────┘
```

## Вкладка Лента

Показывает только обычный пользовательский контент:

```text
GET /api/posts/feed?scope=personal
```

В Ленте не должны хаотично появляться бизнес-посты и предложения.

## Вкладка Витрина

Показывает бизнес-контент:

```text
GET /api/offers/showcase
```

Типы элементов:

```text
business_post — пост бизнес-аккаунта
offer — предложение бизнеса
```

## Создание предложения

Открывается только у бизнес-аккаунта с ролью `owner/admin/smm`:

```text
+ → Что создать? → Предложение → CreateOfferScreen
```

Личный аккаунт не видит пункт `Предложение`.  
Бизнес-роль `messages` не видит пункт `Предложение` и backend запрещает создание.

## Создание предложения: поля

Обязательные:

```text
тип
название
обложка / coverUrl
```

Необязательные:

```text
описание
цена / условие
срок действия
адрес
геометка без карты
```

## Бизнес-профиль

Вкладка бизнес-профиля:

```text
Посты / Видео / Витрина
```

`Витрина` показывает предложения текущего бизнес-аккаунта.

## Экран предложения

Показывает:

```text
обложка
тип
название
бизнес
описание
цена / условие
срок
адрес
```

Нижние действия:

```text
Написать | Маршрут | Позвонить | Сайт | Сохранить
```

Кнопки показываются только если есть нужные данные.

## Backend/API

```text
POST /api/offers
GET /api/offers/my
GET /api/offers/showcase
GET /api/offers/:offerId
POST /api/offers/:offerId/save
GET /api/business/:accountId/offers
GET /api/posts/feed?scope=personal|business
```
