# Близз — страница 8 / Комментарии к посту

## Статус

Реализовано как продолжение версии 7 refresh-fix.

## Смысл

Комментарии открываются из карточки поста в Главной / ленте через светлый bottom sheet. Отдельная страница комментариев не создаётся.

## Переход

```text
Главная → карточка поста → Коммент. N → CommentsSheet(postId)
```

## Кнопки

| Кнопка | Функция | Куда ведёт |
| --- | --- | --- |
| Коммент. N | Открывает bottom sheet комментариев | CommentsSheet |
| Закрыть / overlay | Закрывает комментарии | Главная |
| Отпр. | Создаёт комментарий от activeAccountId | POST /api/posts/:postId/comments |
| Ответить | Подставляет @username в поле ввода | CommentsSheet |
| Удалить | Удаляет комментарий, если есть право | DELETE /api/comments/:commentId |

## Backend

```text
GET /api/posts/:postId/comments
POST /api/posts/:postId/comments
DELETE /api/comments/:commentId
```

Backend определяет `userId`, `activeAccountId` и права по session token. Клиент не отправляет `userId` как источник доверия.

## Права удаления

| Кто | Может удалить |
| --- | --- |
| Автор комментария | Свой комментарий |
| Автор поста | Любой комментарий под своим постом |
| Другой аккаунт | Не может |

## MVP-ограничение ответов

Сложное дерево ответов не реализуется. Кнопка `Ответить` подставляет `@username` в поле ввода.

## Пустое состояние

```text
Пока нет комментариев
```

Без лишних подсказок.

## Не реализовано в этой версии

- лайки комментариев;
- жалобы на комментарии;
- ответы деревом;
- сортировка комментариев;
- комментарии к видео.
