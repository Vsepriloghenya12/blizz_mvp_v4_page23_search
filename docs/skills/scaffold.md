# Scaffold для проекта «Близз»

Источник: рабочий подход из `skill-scaffold`: сначала структура и правила, потом код.

## Перед реализацией модуля

1. Зафиксировать назначение модуля.
2. Зафиксировать, где модуль открывается в навигации.
3. Описать кнопки и переходы.
4. Описать роли и права.
5. Описать backend/API.
6. Описать модели данных.
7. Описать состояния loading/empty/error/noAccess/sessionExpired.
8. Отдельно написать, что запрещено добавлять.
9. Только после этого писать код.

## Архитектурные правила

- Каждый новый модуль — продолжение рабочего приложения, а не отдельный экран.
- Не раздувать один файл без необходимости.
- `User` не равен `Account`.
- Все действия идут через `activeAccountId`.
- Backend не доверяет `userId` с клиента.
- Нельзя использовать `user.isBusiness`.
- Нижнее меню остаётся: `Главная / Видео / + / Карта / Профиль`.
- `Сообщения` не возвращать в нижнее меню.
