# Superpowers для проекта «Близз»

Источник: подход Superpowers — не писать код вслепую, а идти через спецификацию, план, тестирование и проверку.

## Рабочий цикл

1. Brainstorming: уточнить смысл модуля и исключить лишние функции.
2. Writing plans: разложить модуль на API, UI, данные, проверки.
3. TDD/verification: добавить smoke-test для нового backend-сценария.
4. Code review: сверить результат с ТЗ v4 и live-журналом.
5. Completion: приложить архив и честно указать, что проверено.

## Проверки для каждого модуля

- Нет лишних кнопок.
- Нет несогласованных страниц.
- Нет нижней вкладки `Сообщения`.
- Нет кнопки `Обновить` в ленте.
- `Близз` не возвращается в `+ / Что создать?`.
- Бизнес-логика не строится на `user.isBusiness`.
- Доступы проверяются на backend.
- Новый архив содержит все предыдущие модули.
