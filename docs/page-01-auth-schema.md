# Страница 1 — Регистрация / Вход

## Назначение

Первая точка входа в приложение. Страница решает только три задачи:

1. Проверить активную сессию.
2. Зарегистрировать нового пользователя.
3. Войти существующему пользователю.

## Схема переходов

```text
[Запуск приложения]
        |
        v
[GET /api/auth/me]
        |
        |--- session valid ---> [MainTabs / Главная]
        |
        |--- no session ------> [AuthScreen]
                                  |
                                  |--- Создать аккаунт ---> POST /api/auth/register
                                  |                           |
                                  |                           v
                                  |                  User + Personal Account + Session
                                  |                           |
                                  |                           v
                                  |                    [MainTabs / Главная]
                                  |
                                  |--- Войти ------------> POST /api/auth/login
                                                              |
                                                              v
                                                        Session
                                                              |
                                                              v
                                                    [MainTabs / Главная]
```

## Кнопки

| Кнопка | Функция | Переход |
|---|---|---|
| Создать аккаунт | включает режим регистрации | AuthScreen |
| Войти | включает режим входа | AuthScreen |
| Показать пароль | показать/скрыть пароль | без перехода |
| Создать аккаунт, основная | отправляет регистрацию | MainTabs при успехе |
| Войти, основная | отправляет вход | MainTabs при успехе |
| Пользовательское соглашение | будущий правовой экран | пока без перехода |

## Состояния

- `checkingSession`
- `guest`
- `registerMode`
- `loginMode`
- `submitting`
- `fieldError`
- `serverError`
- `success`
- `offline/serverDown`

## Backend/API

- `POST /api/auth/register`
- `POST /api/auth/login`
- `POST /api/auth/logout`
- `GET /api/auth/me`

## Запреты

На странице нет соцлогинов, onboarding, создания бизнеса, интересов, геолокации, уведомлений, выбора города и маркетингового лендинга.
