## 1. Принципы иконографической системы

### 1.1 Сетка и пропорции

- **Базовый размер:** 24×24 pt (соответствует Material и Apple HIG).
- **Контейнер:** 24×24, безопасная зона 20×20 (по 2 pt с каждой стороны).
- **Оптическое выравнивание:** геометрический центр ≠ оптический. Например, галочка «слегка приподнята» вверх, сердце — чуть ниже геометрического центра.
- **Stroke vs fill:** для проекта рекомендую **stroke 1.75 pt с закруглёнными концами** для всех навигационных и контентных иконок. **Filled** — только для активного состояния или для «привытых» семантических (like = filled сердце, save = filled закладка).
- **Скругления концов и углов:** round-cap, round-join. Исключение: технические знаки (close ×, search ⌕) — square-cap.

### 1.2 Состояния иконки

| Состояние   | Параметры                                              |
|-------------|--------------------------------------------------------|
| `default`   | stroke primary text 100%, opacity 1                    |
| `hover`     | opacity 0.85, плавный переход 120ms                    |
| `pressed`  | opacity 0.6, мгновенно                                 |
| `disabled`  | stroke 30% opacity                                     |
| `active`    | filled primary (или stroke primary + soft-blue подложка)|
| `unread`    | filled primary + красная точка-индикатор 8×8           |
| `selected`  | filled primary                                         |

### 1.3 Именование и формат

- Имена — kebab-case, семантические: `IconHome`, `IconHeartFilled`, `IconMessageCircle`.
- Формат: **SVG-компонент** в React Native через `react-native-svg` ИЛИ встроенный `View`+`Text` для ASCII-фасада. Поскольку текущая кодовая база не подключает `react-native-svg`, **первый шаг** — подключить эту зависимость (одна, маленькая, tree-shakeable) и сразу перевести все иконки на SVG. Иначе качества иконок на retina не получить.
- Импорт: `import { IconHome } from '@/shared/ui/icons';` через barrel-экспорт.
- Все иконки — функциональные компоненты с `size: number` и `color: string` props, default 24 / colors.textPrimary.

### 1.4 Правило «одна иконка = одна функция»

- Нельзя использовать «три точки» ⋯ для разных действий (ещё настройки, ещё меню, ещё опции). Если действие другое — иконка другая. Иначе пользователь не различает «настройки» от «меню поста» в туториале.
- Исключение: в `bottom sheet` три точки допустимы, потому что рядом текст.

## 4. Иконки по типу контента на карте и в поиске

> Контекст: `mobile/src/screens/map/MapScreen.tsx:217-222` — сейчас пин-клеймо: P / V / B / Б / ₽. В поиске — текстовые лейблы.

### 4.1 Единый shape language

Каждая иконка контента = **форма + цветной фон**:

| Тип контента | Иконка (24px)                       | Цвет фона (8×8 подложка)  | Текстовая метка |
|--------------|--------------------------------------|---------------------------|-----------------|
| Пост         | `IconImageStack` (две карточки со смещением) | `#EAF1FF` (soft blue)     | «Пост»          |
| Видео        | `IconPlayCircle`                     | `#FFE8E0` (мягкий коралл)  | «Видео»         |
| Близз (сторис) | `IconRingOpen` (незакрытое кольцо) | `#FFE0BD` (мягкий оранж)   | «Близз»         |
| Бизнес       | `IconBriefcase` (портфель)            | `#E5F4E8` (мягкий зелёный)| «Бизнес»        |
| Предложение  | `IconTag` (ценник с дыркой)          | `#FFF1B8` (мягкий жёлтый) | «Предложение»   |

### 4.2 Референсы

- **Foursquare** — классическая карта POI: цветной круглый пин + глиф в центре. Прочитано с любого расстояния.
- **Google Maps** — каплевидный пин с глифом, иконка типа объекта внутри.
- **Apple Maps** — material-стиль, иконка вынесена над каплей.

### 4.3 Рекомендация

Использовать форму **«капля с белым кругом в центре»** (стандартный map pin), внутри белого круга — глиф 16×16 цветом primary. Это сохранит узнаваемость формы пина и одновременно даст color coding для типа.

## 7. Иконка онбординга / empty-state (иллюстративная)

### 7.1 Empty «нет постов»

**Сцена:** `HomeScreen.tsx:1036-1046` — экран Главной с пустой Лентой или Витриной.

**Композиция:** стилизованный квадрат (как карточка) с лёгким tilt 6°, на нём мягкий primary-градиент, над карточкой — лист-бумажка. Цвета — soft blue + primary. Текст: «Пока нет публикаций». Кнопка: `IconPlusCircle` + «Создать пост».

### 7.2 Empty «нет чатов»

**Композиция:** два speech bubbles, пересекающихся в шахматном порядке, один выше, другой ниже. Soft blue + soft orange. Текст: «Пока нет диалогов». Кнопка: «Написать другу».

### 7.3 Empty «нет Близзов»

**Композиция:** кольцо (знак Близза) с маленькой искрой/звёздочкой сверху. Soft orange + primary. Текст: «Близзов пока нет». Кнопка: «Создать Близз».

### 7.4 Empty «Карта пуста»

**Композиция:** карта с одним большим пином-каплей в центре, рядом лупа. Soft blue + primary. Текст: «Здесь пока нет объектов». Кнопка: «Предложить место».

Все иллюстрации — **плоский 2D, линейный стиль с заливкой**, 1-2 цвета, без деталей, которые теряются на 24px. Размер иллюстрации в empty-state: **160×160 pt** в центре экрана.

## 8. Библиотеки-кандидаты

> Контекст: mobile — Expo 56 + react-native-web. Custom SVG-компонент в RN нормально (через `react-native-svg`).

| Библиотека       | Лицензия   | Иконок | Стиль                  | RN support        | Подходит? |
|------------------|------------|--------|------------------------|-------------------|-----------|
| **Phosphor**     | MIT        | 9000+  | 6 weights (thin/thin/regular/light/bold/fill) | через svg      | ✅ лучший выбор — даёт все стили в одной семье, легко переключать active state |
| **Lucide**       | ISC        | 1500+  | stroke 2px, single style | через svg        | ✅ хорошо, простой, но один стиль — для active придётся менять иконку |
| **Tabler**       | MIT        | 4500+  | stroke 1.5/2px           | через svg        | ✅ хорошо, но визуально суше, чем Phosphor |
| **Heroicons**    | MIT        | 1500+  | outline + solid          | через svg        | ⚠️ хороши, но стиль привязан к Tailwind, не всегда ложится |
| **Ionicons**     | MIT        | 1300+  | ios + md                 | через svg        | ⚠️ иконочный iOS стиль, может конфликтовать с нашим |
| **Iconoir**      | MIT        | 1500+  | stroke 1.5px             | через svg        | ✅ похож на Phosphor |
| **SF Symbols**   | proprietary| 5000+  | Apple-only               | web/iOS only     | ❌ нельзя в Android/web build |
| **Custom**       | —          | —      | по ТЗ                    | —                | ⚠️ дорого, медленно, но полный контроль |

### 8.1 Рекомендация: **Phosphor**

Причины:
1. Шесть начертаний одной и той же иконки (`thin`, `light`, `regular`, `bold`, `fill`, `duotone`). Это критично для нашей системы: `active` = `fill`, `default` = `regular`, `disabled` = `light`.
2. Либеральная MIT-лицензия, никаких ограничений для коммерческого использования.
3. SVG-исходники → можно править и экспортировать в наш SVG-компонент.
4. Огромное покрытие (9000+ глифов), включая редкие вроде `Shell`, `Briefcase`, `Tag`.

### 8.2 Альтернатива

Если Phosphor по каким-то причинам не подойдёт — **Iconoir** или **Tabler**. Но в Phosphor уже есть нужные семантические иконки (`Heart`, `HeartFill`, `BookmarkSimple`, `BookmarkSimpleFill`, `PaperPlaneTilt`, `Bell`, `BellFill`, `MagnifyingGlass`, `Plus`, `X`, `CaretLeft`, `DotsThree`, `MapPin`, `Briefcase`, `Tag`, `Tray`, `WarningCircle`, `CheckCircle`, `Info`, `WifiSlash`).

## 10. Открытые вопросы (для пользователя)

1. **Подключаем Phosphor или рисуем свои SVG?** Phosphor — быстро и качественно. Custom — дороже, но полный контроль над пропорциями. (Рекомендую Phosphor как базу с возможностью замены 2-3 иконок на custom.)
2. **Stroke 1.75pt — это финальное решение?** Можно смелее (2.25pt для лучшей видимости на маленьких экранах) или тоньше (1.5pt для более изящного вида).
3. **Filled vs stroke для активного состояния на nav-панели?** Сейчас предлагаю filled. Альтернатива — оставить stroke, но менять цвет на primary + soft-blue подложка (минимально инвазивно).
4. **Что делать с иконкой плюс в центре?** Круг + плюс (как в iOS compose) — стандарт. Альтернатива — квадрат с закруглёнными углами + плюс. Первая — мягче, вторая — современнее.
5. **Empty-state иллюстрации: рисуем сами или берём из unDraw / Storyset?** Рекомендую свои, в стилистике бренда, но unDraw — хороший fallback для прототипа.
