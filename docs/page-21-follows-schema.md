# Page 21 — Подписки, подписчики и заявки

## Назначение

Модуль добавляет социальный граф на уровне `Account`, а не `User`.

Основные действия:

- подписаться на аккаунт;
- отменить подписку;
- получить состояние `Подписаться / Запрошено / Подписки`;
- посмотреть подписчиков;
- посмотреть подписки;
- принять или отклонить заявку закрытого аккаунта;
- учитывать подписку при доступе к закрытому и followers-only контенту.

## Входы

- `Профиль → Подписчики`;
- `Профиль → Подписки`;
- `Настройки и действия → Кто может видеть ваш контент → Заявки на подписку`;
- `Настройки и действия → Подписчики / Подписки`.

## API

- `GET /api/accounts/:accountId/follow-state`
- `POST /api/accounts/:accountId/follow`
- `DELETE /api/accounts/:accountId/follow`
- `GET /api/accounts/:accountId/followers`
- `GET /api/accounts/:accountId/following`
- `GET /api/follow-requests`
- `POST /api/follow-requests/:requestId/accept`
- `POST /api/follow-requests/:requestId/decline`

## Правила доступа

- backend берёт `activeAccountId` из session token;
- `userId` с клиента не принимается как источник прав;
- нельзя подписаться на свой аккаунт;
- открытый аккаунт получает `active` follow сразу;
- закрытый аккаунт получает `pending` request;
- владелец закрытого аккаунта принимает или отклоняет заявку;
- followers-only контент виден только владельцу и accepted followers;
- контент закрытого аккаунта не виден неподписчику даже при `visibility=public`.

## Что не добавлено

- рекомендации людей;
- импорт контактов;
- синхронизация телефона;
- массовые подписки;
- платные подписки;
- подписки авторов;
- рейтинг популярности;
- накрутки;
- случайные демо-пользователи.
