# Page 11 — Близзы / Stories

## Согласованная логика

Близз не создаётся через центральный `+ / Что создать?`. Создание открывается только из строки Близзов на Главной:

```text
Главная → строка Близзов → Ваш Близз + → Создание Близза
```

## Что реализовано

- `Ваш Близз +` открывает `CreateStoryScreen`.
- Близз создаётся через `mediaUrl` и `mediaType` (`image` или `video`) для стабильной web-проверки.
- Близз создаётся от имени `activeAccountId`.
- Личный аккаунт и бизнес `owner/admin/smm` могут создавать Близзы.
- Бизнес-роль `messages` не может создавать Близзы; backend запрещает действие.
- `expiresAt = createdAt + 24 часа`.
- В строке Главной Близзы группируются по аккаунту: один аккаунт = один кружок.
- Просмотр открывается fullscreen-модалкой `StoryViewer`.
- При просмотре фиксируется `view`; повторный просмотр тем же аккаунтом не создаёт дубль.
- Автор видит `viewsCount` в fullscreen-просмотре.
- Ответ на Близз сохраняется через backend как `story_reply`.

## Backend API

```text
POST /api/stories
GET /api/stories/feed
GET /api/stories/my
GET /api/stories/:accountId
POST /api/stories/:storyId/view
POST /api/stories/:storyId/reply
```

## Не реализовано в этом модуле

- Близз в центральном `+`.
- Медиа-picker и выбор из галереи.
- Сложный медиа-редактор.
- Стикеры, опросы, вопросы, музыка, фильтры, AR.
- Highlights.
- Архив Близзов отдельным экраном.
- Список просмотревших отдельным sheet.
- Полноценный чат сообщений для ответов.
- Share для Близзов.
