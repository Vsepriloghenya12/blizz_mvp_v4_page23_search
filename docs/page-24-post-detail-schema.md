# Page 24 — Отдельный экран поста

## Назначение

`PostDetailScreen` открывает конкретный опубликованный пост, доступный текущему `activeAccountId`.

## Входы

- Поиск → результат типа `post`.
- Сохранённое → сохранённый пост.
- Карта → объект типа `post`.
- Главная → фото или текст поста.
- Личный/бизнес-профиль → опубликованный пост.
- Публичный профиль → опубликованный доступный пост.
- Сообщения → карточка отправленного поста.

## UI

- Заголовок `Пост`;
- кнопка назад;
- автор с переходом в публичный профиль;
- фото, текст, геометка и время публикации;
- действия `лайк`, `сохранить`;
- секция комментариев с отправкой и удалением доступных комментариев.

## API

- `GET /api/posts/:postId`
- уже существующие:
  - `POST /api/posts/:postId/like`
  - `POST /api/posts/:postId/save`
  - `GET /api/posts/:postId/comments`
  - `POST /api/posts/:postId/comments`
  - `DELETE /api/comments/:commentId`

## Доступы

Backend определяет viewer по session token и `activeAccountId`. Закрытый и followers-only контент не отдаётся неподписчику.

## Запрещено

- Не добавлять отдельную нижнюю вкладку.
- Не возвращать `Сообщения` в нижнее меню.
- Не возвращать `Близз` в центральный `+`.
- Не добавлять рекомендации, рекламу, AI, рейтинги, похожие посты и новый create-flow.
