# Страница 3 — Профиль личного аккаунта

## Статус

Логика согласована и реализована в архиве `blizz-mvp-v4-page03-profile.zip`.

## Главные решения

- Профиль открывается через `activeAccountId`, а не через `userId`.
- Если активный аккаунт `personal`, показывается личный профиль.
- Верхний счётчик `Посты 0` убран.
- В верхней статистике остались только `Подписчики` и `Подписки`.
- Количество контента показывается во вкладках.
- Блок быстрых разделов `Близзы / Маршруты / Места / Отзывы` убран.
- Подсказка `Создайте первый пост, Близз или видео.` убрана.
- Оставлено нейтральное пустое состояние `Пока нет публикаций`.
- В правом верхнем углу профиля добавлены кнопки личных сообщений и управления профилем.

## Шапка профиля

```text
┌────────────────────────────────────┐
│ Пользователь ▼              ✉  ☰   │
│ @user_123456                       │
└────────────────────────────────────┘
```

| Кнопка | Функция | Куда ведёт |
|---|---|---|
| `Пользователь ▼` | будущий переключатель аккаунтов | позже `AccountSwitcherSheet` |
| `✉` | личные сообщения | `MessagesScreen` |
| `☰` | управление профилем | `ProfileMenuScreen` |

## Верхняя статистика

```text
Подписчики 0        Подписки 0
```

Верхний показатель `Посты 0` не используется, чтобы не дублировать счётчик во вкладках.

## Вкладки

```text
Посты 0 / Видео 0 / Черновики 0 / Сохранённое 0
```

| Вкладка | Что показывает | Кто видит |
|---|---|---|
| `Посты` | опубликованные посты | владелец и разрешённые зрители |
| `Видео` | опубликованные видео | владелец и разрешённые зрители |
| `Черновики` | невыложенные посты | только владелец аккаунта |
| `Сохранённое` | сохранённое пользователя | только владелец аккаунта |

## Редактирование профиля

Отдельный экран, не модалка.

Поля:

- имя;
- никнейм;
- о себе;
- город;
- ссылка.

Ограничения:

- `name`: 1–40 символов;
- `username`: латиница, цифры, подчёркивание, 3–24 символа;
- `bio`: до 150 символов;
- `city`: до 60 символов;
- `link`: до 120 символов.

## API

```text
GET /api/profile/me
PATCH /api/profile/me
```

Backend сам определяет `userId` и `activeAccountId` по session token. Клиент не отправляет `userId` как источник доверия.
