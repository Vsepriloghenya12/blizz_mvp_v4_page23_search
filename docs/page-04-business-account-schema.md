# Страница 4 — создание бизнес-аккаунта и переключение аккаунтов

## Статус

Логика страницы согласована и реализована как продолжение рабочего приложения версии 3.

## Главный принцип

`User` не становится бизнесом. Бизнес создаётся как отдельный `Account type=business`.

После создания бизнеса backend создаёт:

- `Business Account`
- `BusinessProfile`
- `AccountMembership role=owner`
- обновляет `Session.activeAccountId = businessAccountId`

## Основные переходы

```text
Профиль → ☰ Управление профилем → Создать бизнес → POST /api/accounts/business → Бизнес-профиль
```

```text
Профиль → имя аккаунта ▼ → Аккаунты → выбрать аккаунт → POST /api/accounts/switch → Профиль activeAccountId
```

## Поля создания бизнеса

Обязательные:

- Название бизнеса
- Никнейм бизнеса
- Категория

Необязательные:

- Описание
- Адрес
- Телефон
- Сайт / ссылка

Геометка бизнеса не добавлена в этом модуле. Она будет согласовываться в модуле карты и мест.

## Backend/API

```text
POST /api/accounts/business
POST /api/accounts/switch
GET /api/profile/me
```

Backend сам определяет `userId`, `activeAccountId` и роль по session token. Клиент не отправляет `userId` как источник доверия.

## Базовый бизнес-профиль

В версии 4 бизнес-профиль показывает:

- название бизнеса;
- username;
- категорию;
- описание;
- адрес, телефон, сайт, если заполнены;
- статистику: подписчики и публикации;
- вкладки: Посты, Видео, Витрина;
- действия владельца: Редактировать, Добавить предложение, Метрики, Управление.

Эти действия пока визуально присутствуют как будущие рабочие действия бизнес-профиля, но их отдельные страницы будут согласованы позже.

## Запрещено в этом модуле

Не добавлено:

- `user.isBusiness`;
- замена личного аккаунта бизнесом;
- создание бизнеса на Auth-странице;
- создание бизнеса через нижний `+`;
- обязательное создание бизнеса после регистрации;
- сотрудники и роли в интерфейсе;
- карта/геометка бизнеса;
- витрина и предложения как полноценный модуль;
- рейтинг, звёзды, отзывы и внутренние метрики для пользователя.
