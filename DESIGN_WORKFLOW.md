# Design Workflow — Близз

Как работать над новым экраном или модулем в проекте Близз.

---

## Шаг 1. Запрос

Пользователь называет экран или модуль который нужно сделать.

**Пример:**
> "Сделай экран публичного профиля бизнеса"

---

## Шаг 2. Логика экрана

Claude вызывает skill `blizz-screen-flow` и описывает логику:

- Назначение экрана
- Все кнопки и действия
- Переходы (откуда / куда)
- Роли (личный / бизнес / admin)
- Состояния (loading, empty, error, disabled)
- Backend endpoints
- Связи с другими экранами

**⏸️ Пауза — ждём подтверждения логики от пользователя.**

---

## Шаг 3. Визуал

После подтверждения логики — Claude описывает UI через `mobile-ui-designer`:

- Структура (header / main / bottom)
- Карточки, списки, отступы
- Типографика
- Состояния (loading skeleton / empty / error)
- Что запрещено добавлять

Если есть Figma/screenshot → используется как визуальная правда (`figma-to-blizz-screen`).

**⏸️ Пауза — ждём подтверждения визуала от пользователя.**

---

## Шаг 4. Код

После подтверждения — `react-native-ui-implementer` пишет код:

- Читает существующий экран если он есть
- Проверяет backend-связи
- Реализует согласованный UI
- Использует theme.ts
- Не добавляет несогласованные функции

---

## Шаг 5. Design System Check

`design-system-guardian` проверяет:

- Цвета из theme.ts
- Типографика по иерархии
- Кнопки / поля / карточки по стандарту
- Повторяемые паттерны

---

## Шаг 6. Visual QA

`visual-qa-reviewer` проверяет:

- Screenshot / snapshot экрана
- Соответствие референсу (если есть)
- Обрезанные тексты
- Отступы и плотность
- Все состояния работают

---

## Шаг 7. Accessibility

`accessibility-auditor` проверяет:

- Touch targets ≥ 44pt
- accessibilityLabel на кнопках
- Контраст
- Safe area
- Маленький Android (360px)
- Keyboard avoiding

---

## Шаг 8. Финальная проверка

`blizz-final-check` запускает:

```bash
cd mobile && npx tsc --noEmit
grep -r "isBusiness" mobile/src/
grep -r "Сообщения" mobile/src/navigation/MainTabs.tsx
```

Выдаёт отчёт изменённых файлов.

---

## Шаг 9. Отчёт

Claude выдаёт:

```
## Готово: [Экран]

### Изменённые файлы
- mobile/src/screens/.../[Screen].tsx
- mobile/src/navigation/MainTabs.tsx (если добавлен route)
- server/src/modules/.../... (если добавлен endpoint)

### Typecheck ✅
### Grep проверки ✅

### Что сделано
[краткий список]

### Что НЕ сделано (если есть)
[что осталось за рамками задачи]
```

---

## Важные правила процесса

1. **Нельзя пропускать шаги** — логика → визуал → код, не наоборот
2. **Референс = правда** — если есть Figma/screenshot, следовать ему
3. **Продолжать приложение** — не создавать изолированные заглушки
4. **Согласовывать сомнения** — если что-то не описано в ТЗ, спросить
5. **Минимум изменений** — делать только то, что требует задача

---

## Как запустить workflow вручную

```
/blizz-screen-flow [название экрана]
```

После согласования логики скажите "ок, делай визуал" или "делай код".
