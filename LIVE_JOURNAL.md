# Live-журнал Близз MVP v4

## Запись 000 — источник правды

Главный документ: `blizz_full_tz_mvp_detailed_v4`.
Старый код и старые решения не использовать, если они противоречат v4.

## Запись 001 — порядок разработки

Проект собирается как единое рабочее приложение.
Каждая новая страница добавляется внутрь уже существующего приложения.
После реализации страницы выдаётся новый архив проекта.
Сначала логика, финальный визуал — позже.

## Запись 002 — формат страницы

Каждая страница перед кодом фиксируется как схема:

- назначение;
- кто видит;
- блоки страницы;
- кнопки;
- куда ведёт каждая кнопка;
- состояния;
- backend/API;
- связи с другими страницами;
- запреты на лишние кнопки и дубли.

## Запись 003 — страница 1: Регистрация / Вход

Согласовано:

- одна общая Auth-страница с переключателем `Создать аккаунт / Войти`;
- поля только `телефон/email` и `пароль`;
- кнопка показать/скрыть пароль;
- ссылка на пользовательское соглашение;
- при запуске приложения сначала `GET /api/auth/me`;
- если session валидна — `MainTabs / Главная`;
- если session нет — `AuthScreen`;
- регистрация вызывает `POST /api/auth/register`;
- backend создаёт `User`, `Personal Account`, `AccountMembership`, `Session`;
- `activeAccountId = personalAccountId`;
- вход вызывает `POST /api/auth/login`;
- после успеха — сразу `MainTabs`, без onboarding.

Запрещено на странице:

- соцлогины;
- выбор интересов;
- выбор города;
- обязательная геолокация;
- обязательные уведомления;
- создание бизнеса;
- заполнение профиля;
- переключатель личный/бизнес;
- маркетинговый лендинг;
- лишние блоки.

## Запись 004 — Навигация / MainTabs

Дата: 2026-06-13.

Пользователь изменил структуру нижнего меню относительно исходного v4.

Зафиксировано:

```text
Главная / Видео / + / Карта / Профиль
```

Сообщения больше не являются пунктом нижнего меню. Вход в сообщения расположен в правом верхнем углу шапки Главной. Сообщения остаются полноценным модулем: личные чаты, группы, бизнес-чаты, shared_content, игры и ответы на Близзы.

Центральный плюс открывает экран “Что создать?”, а не создаёт контент напрямую.

Профиль всегда открывает activeAccountId. Запрещено строить профиль по user.isBusiness.

Реализация архива v2 должна быть продолжением страницы 1, а не отдельным набором экранов.

## Запись 004 — изменение нижнего меню и схема MainTabs

Решение пользователя изменяет исходную структуру v4: нижнее меню теперь `Главная / Видео / + / Карта / Профиль`.

Сообщения больше не пункт нижнего меню. Основной вход в сообщения находится в правом верхнем углу шапки Главной. Сообщения остаются полноценным модулем: личные чаты, группы, бизнес-чаты, shared_content, ответы на Близзы и игры.

Правила:
- `+` открывает экран “Что создать?”, а не создаёт контент напрямую.
- `Видео` открывает видеоленту, но создание видео идёт через `+`.
- `Карта` остаётся в нижнем меню как ключевая функция Близз.
- На карте нет кнопки создания контента.
- `Профиль` открывает `ProfileScreen(activeAccountId)`.
- Страницы остаются связанными внутри одного рабочего приложения, а не отдельными демо-экранами.

Фиксации:

- вкладка `Черновики` предназначена для невыложенных постов владельца аккаунта;

- редактирование профиля сделано отдельным экраном;
- меню профиля открывает только уже связанные действия, без полного экрана настроек до отдельного согласования.

Backend:

## Запись 010 — страница 4: создание бизнес-аккаунта и переключение аккаунтов

Реализована версия 4 как продолжение версии 3. Добавлено создание бизнес-аккаунта из управления профилем и из переключателя аккаунтов. Backend endpoint `POST /api/accounts/business` создаёт отдельный `Account type=business`, `BusinessProfile`, `AccountMembership role=owner` и переключает `Session.activeAccountId` на созданный бизнес. Личный аккаунт не заменяется бизнесом. Добавлен UI переключения аккаунтов через имя аккаунта с chevron в профиле. `POST /api/accounts/switch` подключён к UI. Профиль теперь открывает личный или бизнес-профиль в зависимости от `activeAccount.type`. Витрина, предложения, сотрудники, геометка бизнеса и метрики бизнеса пока не реализованы как отдельные модули, чтобы не добавлять несогласованные функции.

## Запись 011 — создание Близза убрано из `+`

Центральный `+` больше не показывает создание Близза. Близз будет создаваться из блока Близзов пользователя, как stories-механика. Экран `+ / Что создать?` теперь:

- личный аккаунт: Пост / Видео;
- бизнес owner/admin/smm: Пост / Видео / Предложение;
- business messages: состояние «У вас доступ только к сообщениям» + «Открыть сообщения».

## Запись 012 — страница 5 `+ / Что создать?` реализована

Экран `+ / Что создать?` реализован как продолжение версии 4. Убран пункт «Близз». Добавлен кликабельный активный аккаунт с переходом в переключатель аккаунтов. Пункты создания зависят от `activeAccount.type` и роли. Для роли `messages` отображается только доступ к сообщениям. Пункты `Пост`, `Видео`, `Предложение` ведут на короткие intro-экраны будущих модулей без публикации и редакторов, чтобы не добавлять несогласованные функции. Полные страницы создания будут согласованы отдельно.

## Запись 013 — страница 6: создание поста реализовано

## Запись 015 — Страница 8: Комментарии к посту

Реализованы комментарии к постам как продолжение версии 7 refresh-fix. Комментарии открываются из карточки поста в Главной через светлый bottom sheet, отдельный экран комментариев не добавлен. Ответы реализованы MVP-логикой через подстановку `@username` в поле ввода, без дерева ответов. Пустое состояние: «Пока нет комментариев». Лайки комментариев, жалобы и дерево ответов вынесены в будущие модули.

## Запись 016 — Модуль 9: Поделиться / Share

Модуль Share реализован как продолжение версии 8 comments. Кнопка `Поделиться` в карточке поста на Главной открывает светлый bottom sheet, отдельный экран Share не добавлен. Быстрые получатели — только собственные аккаунты пользователя, без случайных демо-получателей. Добавлены действия `Скопировать ссылку` и `Поделиться вне приложения`. ## Запись 019 — Модуль 12: Главная / Лента и Витрина

Пользователь уточнил логику Главной: у обычного пользователя на Главной должны быть две вкладки/режима: `Лента` и `Витрина`. Это не отдельная вкладка нижнего меню и не случайный блок `Предложения рядом`.

Реализовано как продолжение версии 11 Stories:
- Главная теперь имеет переключатель `Лента / Витрина` внутри экрана `Главная`.
- `Лента` показывает только опубликованные посты личных аккаунтов.
- Бизнес-посты больше не смешиваются с обычной пользовательской лентой.
- `Витрина` показывает бизнес-контент: посты бизнес-аккаунтов и предложения.
- Центральный `+` для бизнес `owner/admin/smm` открывает `Предложение` и теперь ведёт на рабочий `CreateOfferScreen`, а не intro-экран.
- Личный аккаунт не видит `Предложение` в `+`.
- Бизнес-роль `messages` не может создавать предложения, backend запрещает это.
- В бизнес-профиле вкладка `Витрина` показывает предложения текущего бизнес-аккаунта.
- Добавлен экран предложения `OfferDetailScreen` с действиями `Написать`, `Маршрут`, `Позвонить`, `Сайт`, `Сохранить`; кнопки показываются по данным предложения/бизнеса.
- Предложение сохраняется через `POST /api/offers/:offerId/save`.

Backend:
- добавлены `POST /api/offers`, `GET /api/offers/my`, `GET /api/offers/showcase`, `GET /api/offers/:offerId`, `POST /api/offers/:offerId/save`, `GET /api/business/:accountId/offers`;
- `GET /api/posts/feed` теперь принимает `scope=personal|business`; в Главной вкладка `Лента` использует `scope=personal`;
- `GET /api/offers/showcase` отдаёт смешанную Витрину: `business_post` и `offer`.

Правила:
- нижнее меню остаётся `Главная / Видео / + / Карта / Профиль`;
- отдельной нижней вкладки `Витрина` нет;
- блока `Предложения рядом` на Главной нет;
- бизнес-контент не смешивается в обычную `Ленту`;
- не добавлены оплата, корзина, доставка, склад, рейтинг, звёзды, лайки/комментарии предложений и маркетплейс-логика.

Проверки:
- server `npm run check`;
- server `page12-smoke-test.js`;
- mobile `npm install`;
